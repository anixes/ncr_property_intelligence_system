## 🧠 GSD Methodology (Get Shit Done)

Every task MUST follow the atomic lifecycle:
1.  **SPEC**: Define exactly what we are building.
2.  **PLAN**: Break it into small, dependency-aware tasks.
3.  **EXECUTE**: Implement task-by-task with **Atomic Git Commits**.
4.  **VERIFY**: Show empirical evidence (tests, CLI output) that it works.

**Guardrails:**
*   **NO Placeholders**: Use `generate_image` or real data.
*   **NO "Vibecoding"**: If a feature is complex, perform level-2 research first.
*   **Empirical Proof**: If a test fails, the task is NOT complete.
